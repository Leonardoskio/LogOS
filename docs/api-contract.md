# API Contract MVP

This document is the programming contract between frontend and backend for the
first working version of LogOS.

The field-level source of truth is:

```text
docs/data-model.md
```

Reference request payloads are stored in:

```text
examples/
```

Base URL during local development:

```text
http://localhost:3000/api
```

## General Rules

All requests and responses use JSON.

```text
Content-Type: application/json
```

The frontend must not write directly to the Obsidian vault. It only calls the
backend.

The backend is responsible for:

```text
validating request data
normalizing IDs
generating official shipment IDs
creating archive folders
rendering Markdown templates
writing notes into the active vault
returning stable JSON responses
```

Timestamps such as `created_at` and `updated_at` are generated by the backend.
The frontend should not send them during create operations.

## Error Format

All API errors should use this shape:

```json
{
  "error": {
    "code": "missing_required_field",
    "message": "Missing required field: cliente_nome",
    "field": "cliente_nome"
  }
}
```

`field` is optional and should be present when the error refers to a specific
request field.

Common error codes:

```text
400 missing_required_field
400 invalid_date
400 invalid_tipo_carico
400 invalid_stato
400 invalid_number
404 not_found
409 duplicate_id
500 vault_read_failed
500 vault_write_failed
500 template_render_failed
```

## Shipments

### Create Shipment

```text
POST /api/spedizioni
```

Creates one shipment note in the active company vault.

Reference payload:

```text
examples/spedizione-create.json
```

Request body:

```json
{
  "data_operativa": "2026-05-27",
  "cliente_id": "azienda-bianchi",
  "cliente_nome": "Azienda Bianchi",
  "merce": "Liquido alimentare",
  "tipo_carico": "LQ",
  "origine": "Centro operativo",
  "destinazione": "Via Roma 12, Milano",
  "autista_id": "mario-rossi",
  "motrice_id": "AB123CD",
  "distanza_km": 230,
  "note": "Scarico entro le 10:00"
}
```

Required request fields:

```text
data_operativa
cliente_id
cliente_nome
merce
tipo_carico
origine
destinazione
```

Optional request fields:

```text
autista_id
motrice_id
distanza_km
note
stato
```

Default values:

```text
stato = richiesta
```

Backend behavior:

```text
1. validate required fields
2. validate data_operativa
3. validate tipo_carico against docs/data-model.md
4. validate stato if present
5. generate id as SP-YYYY-NNNNN
6. generate created_at and updated_at
7. create archive path from data_operativa
8. render vault-template/00 Template/Spedizione.md
9. write Markdown note under 02 Archivio
10. return created note metadata
```

Success response:

```json
{
  "data": {
    "type": "spedizione",
    "id": "SP-2026-00001",
    "data_operativa": "2026-05-27",
    "cliente_id": "azienda-bianchi",
    "cliente_nome": "Azienda Bianchi",
    "tipo_carico": "LQ",
    "stato": "richiesta",
    "file": "SP-2026-00001_azienda-bianchi_LQ.md",
    "path": "02 Archivio/2026/Maggio/27/SP-2026-00001_azienda-bianchi_LQ.md"
  }
}
```


## Customers

### Create Customer

```text
POST /api/clienti
```

Reference payload:

```text
examples/cliente-create.json
```

Request body:

```json
{
  "nome": "Azienda Bianchi",
  "riferimento": "Luca Bianchi",
  "telefono": "+39 000 000000",
  "email": "logistica@bianchi.example",
  "sede_citta": "Milano",
  "sede_provincia": "MI",
  "sede_cap": "20100",
  "sede_via": "Via Roma",
  "sede_numero_civico": "12"
}
```

Required request fields:

```text
nome
```

Optional request fields:

```text
riferimento
telefono
email
sede_citta
sede_provincia
sede_cap
sede_via
sede_numero_civico
```

Backend behavior:

```text
1. validate nome
2. generate cliente id from nome
3. add suffix if id already exists
4. generate created_at and updated_at
5. render vault-template/00 Template/Cliente.md
6. write note under 01 Home/Clienti
```

Success response:

```json
{
  "data": {
    "type": "cliente",
    "id": "azienda-bianchi",
    "nome": "Azienda Bianchi",
    "path": "01 Home/Clienti/azienda-bianchi.md"
  }
}
```

### List Customers

```text
GET /api/clienti
```

Success response:

```json
{
  "data": [
    {
      "type": "cliente",
      "id": "azienda-bianchi",
      "nome": "Azienda Bianchi",
      "spedizioni_count": 1,
      "path": "01 Home/Clienti/azienda-bianchi.md"
    }
  ]
}
```

`spedizioni_count` is derived from shipment notes. It is not stored as primary
customer data.

### Read Customer

```text

```

Success response:

```json

  "data": {
    "type": "cliente",
    "id": "azienda-bianchi",
    "frontmatter": {},
    "markdown": "# Cliente Azienda Bianchi..."
  }
}
```

### List Customer Shipments

```text
GET /api/clienti/:id/spedizioni
```

Equivalent filter:

```text
GET /api/spedizioni?cliente_id=:id
```

### Update Tractor Status

```text
PATCH /api/motrici/:id/stato
```

Request body:

```json
{
  "stato": "in_viaggio",
  "posizione": "Milano"
}
```

For the MVP, this endpoint may update a snapshot in the Markdown note. Frequent
real-time GPS updates are out of scope and should not be written continuously to
the vault.

## AI

### Daily Report

```text
POST /api/ai/report-giornaliero
```

Request body:

```json
{
  "data": "2026-05-27"
}
```

Backend behavior:

```text
1. read shipments for the requested data
2. pass structured data to aiService
3. use mockAiProvider during MVP development
4. return Markdown report text
```

Success response:

```json
{
  "data": {
    "type": "report_giornaliero",
    "data": "2026-05-27",
    "markdown": "# Report giornaliero - 2026-05-27..."
  }
}
```

### Logistics Suggestions

```text
POST /api/ai/consigli-logistici
```

Request body:

```json
{
  "periodo": "2026-05"
}
```

AI endpoints read structured data already created by the backend. AI must not
generate official IDs, choose archive paths or overwrite primary logistics
records.

## MVP Demo Flow

The first end-to-end demo should use this order:

```text
1. GET /api/health
2. POST /api/clienti
3. POST /api/autisti
4. POST /api/motrici
5. POST /api/spedizioni
6. GET /api/clienti/:id/spedizioni
7. POST /api/ai/report-giornaliero
```

This proves the actual project idea:

```text
form data -> backend -> vault Markdown -> linked views -> AI report
```

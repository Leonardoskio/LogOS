# AI Plan

AI in LogOS is an analysis layer. It should not create primary logistics facts
without backend validation.

## What AI Should Do

AI can:

```text
summarize daily shipments
detect risky or incomplete shipment data
suggest logistics optimizations
compare customer activity over time
explain tractor utilization
produce human-readable reports
```

## What AI Should Not Do

AI should not:

```text
generate official shipment IDs
choose archive paths
silently overwrite vault notes
invent missing operational data
be the only validator of form input
```

## Data Flow

```text
Form data
  -> backend validation
  -> Markdown note in vault
  -> AI reads structured notes
  -> report or suggestion
```

## Prompt Inputs

Prompts should receive structured data, not raw vault dumps when possible.

Example input shape:

```json
{
  "periodo": "2026-05-27",
  "spedizioni": [
    {
      "id": "SP-2026-00001",
      "cliente_id": "bianchi",
      "tipo_carico": "LQ",
      "motrice_id": "AB123CD",
      "stato": "completata"
    }
  ]
}
```

## AI Outputs

Reports generated by AI should be saved under:

```text
vaults/<company>/03 AI/
```

Every generated report should mention:

```text
period covered
source shipment IDs
key findings
uncertainties or missing data
recommended actions
```

## Providers

Initial development should use `mockAiProvider.js`. A real provider can be added
later behind `aiService.js` without changing route contracts.
